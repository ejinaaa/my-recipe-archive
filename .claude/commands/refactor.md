# 리팩터링 워크플로우

사용자가 지정한 코드를 분석하고 리팩터링을 수행합니다.

## 워크플로우

### 1단계: 분석

대상 코드를 읽고 다음을 분석합니다:

- 현재 코드 구조 파악
- 컴포넌트/함수 크기 (줄 수)
- 책임 분리 상태
- 중복 코드 여부
- FSD 레이어 적합성

### 2단계: 문제점 보고

발견된 문제점을 정리하여 보고합니다:

```
## 분석 결과

### 현재 상태
- 파일: [파일 경로]
- 크기: [줄 수]
- 레이어: [현재 레이어]

### 발견된 문제
1. [문제 1]: [설명]
2. [문제 2]: [설명]
...
```

### 3단계: 리팩터링 제안

구체적인 리팩터링 방안을 제시합니다:

```
## 리팩터링 제안

### 방안 1: [제안명]
- 변경 내용: [설명]
- 예상 결과: [설명]

### 방안 2: [제안명]
- 변경 내용: [설명]
- 예상 결과: [설명]
```

### 4단계: 사용자 확인

사용자에게 어떤 방안으로 진행할지 확인합니다.

### 5단계: 구현

승인된 방안에 따라 리팩터링을 수행합니다:

1. 코드 분리/수정
2. FSD 구조에 맞게 파일 배치
3. import 경로 업데이트
4. 기존 코드와의 호환성 확인

### 6단계: 검증

리팩터링 완료 후:

1. `pnpm build` 실행하여 빌드 확인
2. 에러가 있으면 수정
3. 변경 사항 요약 보고

## 주의사항

- 기존 기능이 유지되도록 함
- 한 번에 너무 많은 변경을 하지 않음
- 각 단계별로 사용자 확인을 받음
- FSD 아키텍처 규칙 준수
- 프로젝트 코드 스타일 유지
