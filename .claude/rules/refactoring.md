# 리팩터링 원칙

## 컴포넌트 분리 기준

- **150줄 이상**
- **2개 이상의 책임**
- **2회 이상 반복**되는 로직
- **복잡한 조건부 렌더링**

## 함수 분리 기준

- **20줄 이상**
- **하나의 작업만** 수행
- 함수명으로 **역할이 명확**하게

## 커스텀 훅 추출 기준

- 상태 로직이 컴포넌트에서 **분리 가능**
- 같은 상태 로직이 **여러 컴포넌트에서 사용**
- 컴포넌트의 **관심사 분리** 필요

## FSD 레이어 이동

| 현재 위치 | 이동 조건 | 이동 위치 |
|----------|----------|----------|
| 컴포넌트 내부 유틸 | 2곳 이상에서 사용 | `shared/lib/` |
| features 컴포넌트 | 복잡한 상태 관리 추가 | `widgets/` |
| 컴포넌트 내부 타입 | 여러 파일에서 사용 | `model/types.ts` |
