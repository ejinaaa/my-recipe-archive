# 빌드 검증 및 자동 커밋 통합 워크플로우

빌드를 통해 코드의 안정성을 먼저 검증하고, 성공 시에만 커밋 메시지 작성 및 커밋을 진행한다.

## 1단계: 빌드 검증 (Safety Check)

- pnpm을 사용해서 `build` 스크립트를 실행해.
- **에러 분석**: 빌드 중 TypeScript나 Lint 에러 발생 시, 해당 파일로 이동해 원인을 분석하고 즉시 수정안을 제시해.
- **환경 확인**: `.env` 설정 누락이나 Node.js 버전 문제로 인한 실패 가능성도 함께 체크해.
- **성능 체크**: 빌드 성공 시, 비정상적으로 큰 청크(Chunk)가 있다면 코드 분할(Code Splitting)이나 최적화 방안을 제안해.
- 에러가 있다면 수정안을 제시하고, 내가 수정을 완료할 때까지 커밋 단계로 넘어가지 마.

## 2단계: 커밋 메시지 규칙 (필수 준수)

분석된 변경 사항을 바탕으로 아래 규칙에 따라 메시지를 작성해.

1. **포맷**: `type(scope): subject` 형식을 지킬 것.
2. **타입 구분**:
   - `feat`: 새로운 기능 추가
   - `fix`: 버그 수정
   - `refactor`: 코드 리팩토링 (성능 개선 포함)
   - `docs`: README.md 등 개발 문서의 수정 및 추가
   - `config`: .cursor/rules, .cursor/commands, 라이브러리 설정 등 프로젝트 환경 설정 변경
   - `style`: 코드 포맷팅, 세미콜론 누락 등 (로직 변경 없음)
   - `chore`: 빌드 업무 수정, 패키지 매니저 설정, 기타 작업
3. **언어**: 제목과 상세 설명은 모두 **한글**로 작성할 것.
4. **내용**: '왜(Why)' 변경했는지 맥락을 포함하고, `config` 타입은 어떤 설정이 바뀌었는지 구체적으로 명시할 것.

## 3단계: 실행

- 생성된 메시지를 먼저 보여주고 내 승인을 받아.
- 승인 시 `git commit -m "{메시지}"`를 실행해.
