# 전문적인 Git 커밋 메시지 작성

현재 스테이징된 변경 사항(Staged Changes) 또는 현재 작업 중인 `git diff`를 분석해서 커밋 메시지를 작성해줘.

## 커밋 메시지 규칙

1. **포맷**: `type(scope): subject` 형식을 지킬 것.
2. **타입 구분 (세분화)**:
   - `feat`: 새로운 기능 추가
   - `fix`: 버그 수정
   - `refactor`: 코드 리팩토링 (성능 개선 포함)
   - `docs`: README.md 등 개발 문서의 수정 및 추가
   - `config`: .cursor/rules, .cursor/commands, 라이브러리 설정 등 프로젝트 환경 설정 변경
   - `style`: 코드 포맷팅, 세미콜론 누락 등 (로직 변경 없음)
   - `chore`: 빌드 업무 수정, 패키지 매니저 설정, 기타 자잘한 작업
3. **언어**: 제목과 상세 설명은 모두 **한글**로 작성해줘.
4. **내용**: 왜(Why) 이 변경이 필요한지, 어떤 컴포넌트나 로직에 영향을 주었는지 구체적으로 포함해줘. 특히 `config` 타입의 경우 어떤 설정이 변경되었는지 명시해줘.

메시지를 먼저 보여주고, 내가 확인하면 `git commit -m "{메시지}"` 명령어를 실행해줘.
